# AutoOutlook Deployment Instructions

This project is deployed to Cloudflare Pages with static API artifacts generated by GitHub Actions. Production should not depend on Google Cloud billing, Cloud Run, or Cloud Run jobs.

## Current Production Setup

- Production site: `https://autooutlook.tech`
- WWW domain: `https://www.autooutlook.tech`
- Cloudflare Pages project: `autooutlook-pages`
- GitHub repo: `ShianMike/AutoOutlook`
- Default branch: `master`
- Production refresh workflow: `.github/workflows/free-hosting-refresh.yml`
- Workflow name: `Refresh static AutoOutlook artifacts`
- Schedule: `30 * * * *`, every hour at minute `30` UTC

Important: Cloudflare Pages automatic production deployments are disabled on purpose. Do not re-enable them.

Cloudflare's normal Git build only runs `npm run build`, so it does not include generated `_api` forecast artifacts. If automatic Cloudflare deployments are enabled, a normal commit can overwrite the working artifact deployment and cause missing API files.

## Correct Deployment Path

Use the GitHub Actions workflow for production deploys.

Manual production refresh:

```powershell
gh workflow run "Refresh static AutoOutlook artifacts" --repo ShianMike/AutoOutlook -f force_deploy=true
```

Watch the run:

```powershell
gh run list --repo ShianMike/AutoOutlook --workflow "Refresh static AutoOutlook artifacts" --limit 3
gh run watch <run-id> --repo ShianMike/AutoOutlook --exit-status
```

Inspect failed logs:

```powershell
gh run view <run-id> --repo ShianMike/AutoOutlook --log-failed
```

The artifact generation step can take about 30 minutes on a cold GitHub runner.

## What The Workflow Does

The workflow:

1. Installs Python and Node dependencies.
2. Bootstraps runtime hazard models with `python -m backend.ml.bootstrap_models`.
3. Detects the latest complete HRRR cycle requiring `f48`.
4. Checks `https://autooutlook.tech/api/outlook/incremental`.
5. Skips if production already has the same complete cycle, unless `force_deploy=true`.
6. Generates `f00` through `f48` incremental artifacts.
7. Builds the frontend.
8. Exports static API files into `dist/_api`.
9. Deploys `dist` to Cloudflare Pages production branch `master`.

## When To Use `force_deploy=true`

Use `force_deploy=true` when:

- You changed frontend code and need it live immediately.
- You changed API routing, Pages Functions, or static export code.
- You changed deployment configuration.
- Production was accidentally overwritten by a plain Cloudflare Git deployment.
- You need to restore the full `_api` artifact tree.

Without `force_deploy=true`, the workflow may skip if the current deployed HRRR cycle is already complete.

## Do Not Use These For Production

Do not use Cloudflare Pages automatic Git deployment for production.

Do not deploy with only:

```powershell
npm run build
npx wrangler deploy
```

Do not use the Cloudflare dashboard "Retry deployment" on a plain source build if artifacts are needed.

Do not re-enable:

```text
Cloudflare Pages > autooutlook-pages > Settings > Build > Branch control > Automatic deployments
```

It must remain disabled for production.

## Verification After Every Deploy

Run these checks after every production deployment:

```powershell
$urls = @(
  "https://autooutlook.tech/",
  "https://www.autooutlook.tech/",
  "https://autooutlook.tech/gif.worker.js",
  "https://autooutlook.tech/api/health",
  "https://autooutlook.tech/api/forecast",
  "https://autooutlook.tech/api/outlook/incremental",
  "https://autooutlook.tech/api/outlook/incremental/hour/0/probability-tile",
  "https://autooutlook.tech/api/outlook/incremental/hour/48/risk-polygons"
)

foreach ($url in $urls) {
  $r = Invoke-WebRequest -Uri $url -UseBasicParsing -TimeoutSec 60
  [pscustomobject]@{
    Url = $url
    Status = [int]$r.StatusCode
    ContentType = ($r.Headers["content-type"] -join ",")
    Cors = ($r.Headers["access-control-allow-origin"] -join ",")
  }
}
```

Verify all per-hour API artifacts:

```powershell
$results = @()
foreach ($h in 0..48) {
  foreach ($kind in @("metadata", "probability-tile", "risk-polygons")) {
    $url = "https://autooutlook.tech/api/outlook/incremental/hour/$h/$kind"
    try {
      $r = Invoke-WebRequest -Uri $url -UseBasicParsing -TimeoutSec 30
      $results += [pscustomobject]@{ Hour = $h; Kind = $kind; Status = [int]$r.StatusCode }
    } catch {
      $resp = $_.Exception.Response
      $results += [pscustomobject]@{ Hour = $h; Kind = $kind; Status = if ($resp) { [int]$resp.StatusCode } else { $null } }
    }
  }
}

$bad = $results | Where-Object Status -ne 200
if ($bad) { $bad | Format-Table -AutoSize } else { "all 147 per-hour API checks returned 200" }
```

Verify the index is complete:

```powershell
$idx = Invoke-RestMethod "https://autooutlook.tech/api/outlook/incremental" -TimeoutSec 30
[pscustomobject]@{
  Cycle = $idx.cycle
  Status = $idx.status
  Ready = $idx.readyForecastHours.Count
  Latest = $idx.latestReadyForecastHour
}
```

Expected:

- `Status`: `complete`
- `Ready`: `49`
- `Latest`: `48`

Verify CORS:

```powershell
$r = Invoke-WebRequest -Uri "https://autooutlook.tech/api/health" -Method OPTIONS -UseBasicParsing -TimeoutSec 30
[pscustomobject]@{
  Status = [int]$r.StatusCode
  AllowOrigin = ($r.Headers["access-control-allow-origin"] -join ",")
  AllowMethods = ($r.Headers["access-control-allow-methods"] -join ",")
}
```

Expected:

- `Status`: `204`
- `AllowOrigin`: `*`
- `AllowMethods`: includes `GET, HEAD, OPTIONS`

## Known Failure Mode

Symptom:

```text
GET https://autooutlook.tech/api/outlook/incremental/hour/13/probability-tile 404
GET https://autooutlook.tech/api/outlook/incremental/hour/13/metadata 404
```

Often this repeats for many hours, while `/api/outlook/incremental` still says `complete`.

Root cause:

Cloudflare automatic production deployment overwrote the artifact deployment with a plain frontend build. The good deployment URL may still work, but `autooutlook.tech` points to the newer broken production deployment.

Fix:

1. Disable Cloudflare automatic production deployments.
2. Restore production with either:
   - Roll back to the latest known-good artifact deployment in Cloudflare Pages, or
   - Run the GitHub Actions workflow with `force_deploy=true`.
3. Re-run the full per-hour verification loop above.

## Emergency Rollback

Use Cloudflare dashboard:

```text
Workers & Pages > autooutlook-pages > Deployments
```

Open the most recent known-good artifact deployment, then:

```text
Manage deployment > Rollback to this deployment > Roll back
```

After rollback, immediately verify all API routes. Do not leave automatic deployments enabled, or the next source commit can break production again.

## Normal Code Change Procedure

1. Make code changes.
2. Run local checks:

```powershell
npm run build
python -m py_compile backend\ml\bootstrap_models.py backend\ml\inference.py scripts\detect-hrrr-cycle.py scripts\export-static-api.py
```

3. Commit and push to `master`.
4. Trigger the GitHub Actions workflow with `force_deploy=true`.
5. Watch the run until success.
6. Run all verification checks in this file.

## Production Ownership Rule

Production belongs to the GitHub Actions artifact workflow, not to Cloudflare's automatic Git builder.

If production is broken after a commit, check Cloudflare first for an accidental automatic deployment before debugging frontend code.
